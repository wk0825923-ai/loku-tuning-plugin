EMBEDDED IN LOKU — 機能統合マップ

Tuningは新機能じゃない。
Lokuの既存機能をぜんぶ実名の視線でつなぐ層。

Lokuには既に「トラッキング／流入経路／分析」がある。Tuningはそれらの解像度を上げ、匿名→実名の結合で配信・EC・Mallまで一本の因果でつなぐ。

既存の計測系に"増築"する（重複させない）

Lokuの トラッキング／流入経路／分析 は今は主に集計。Tuningはボックス単位の視線＋実名紐付けを足して、同じ画面の解像度を上げる。新タブは作らない。

LIFF識別が"結合レール"（発明不要）

ダッシュボードの警告「リッチメニューからの友だち識別が無効（LIFF未登録）」＝まさに匿名→実名問題。Lokuの既存LIFF識別基盤に乗るだけで結合できる。

Attention Layer ＝ 匿名の視線を、実名の友だちに結合する背骨

Lokuが配る全ページに計測SDKを効かせ、LINE追加の瞬間に友だちへ結合。以降のLoku機能がその情報で動く。

来る前：検索（サチコ無料API）→ 来た後：ページ視線（自前SDK）→ 結合：LIFFで友だちに→ その後：配信・予約・EC・Mall
既存の計測を強化（解像度UP） タグ発火の供給先（自動化が回る） 計測対象を拡張（EC/Mall/コミュニティ） 直接は絡まない ★＝特に効く

🛒EC商品ページ → カゴ落ち自動化

box＝商品画像／説明／価格／レビュー／カート。「価格で3回戻った」「レビュー熟読で未購入」→ カゴ落ちタグ → リマインダー＋クーポンが自動発火。

🎯クラファン → 迷いリマインド

box＝リターン一覧／ストーリー／進捗。「このリターンで長く迷った」を検知 → 締切前に該当リターンを名指しでリマインド。迷いの可視化が支援額に直結。

📈トラッキング/分析 → 実データ化

今は集計だけの分析系に、実名×ボックス視線を裏打ち。「誰が・どの検索で来て・どこを見て・買ったか」が一本で追える。

🎓コミュニティのコース → 学習Attention

教材ページのどのブロックで離脱したかを計測。つまずき箇所にフォロー配信、完走率を上げる。物販以外にも効く汎用性。

実装上は page.kind を増やすだけ（lp / ec_product / mall_listing / crowdfunding / community_course / form …）。スキーマは schema.sql、設計は ARCHITECTURE.md 参照。
